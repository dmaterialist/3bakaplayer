Плеер плейлистов или 3baka! music player

Цель 3baka! music player - хранить плейлисты из youtube, spotify, яндекс музыки в отдельных списках, позволять выбирать слушать эти плейлисты, позволять редактировать эти списки и названия плейлистов и искать в этих списках.  Для управления приложением используются кнопки, пункты меню и поле поиска:

1. Добавление плейлистов

Для начала работы с 3baka! music player необходимо добавить в него несколько плейлистов. Для этого необходимо нажать на пункт меню "добавить плейлист".  После этого в диалоговом окне необходимо ввести ссылку на плейлист.

Ссылка на плейлист - это любая рабочая ссылка, из которой можно извлечь данные о плейлисте. Например,  https://www.youtube.com/playlist?list=PLxB9p8WAdj0L5LT_UYU_fTmtGmqj8NdVW  тоже распознаётся как правильная ссылка на плейлист, хотя она ведёт на видео в плейлисте.  Для spotify и яндекс.музыки возможно также добавление ссылок на альбомы/подкасты.

Но ссылки на несуществующие плейлисты, на другие сайты, на неработающие сайты, не ссылки будут отклонены. Для проверки существования плейлистов используется получение заголовка веб-страницы с помощью JSoup. По этой причине добавление плейлистов без подключения к Интернету невозможно.  Об ошибке ввода сообщит всплывающее диалоговое окно.

Заголовок веб-страницы используется для создания названия по умолчанию. Его можно изменить прямо в окне добавления плейлиста. 

Невозможно добавление двух одинаковых плейлистов. Это бессмыссленно, учитывая то, что окна с одним плейлистом можно запускать сколько угодно раз.

2.  Проигрывание плейлистов

Плейлист открывается через WebView в iframe в новом окне в углу экрана после нажатия на кнопку "запуск".  В iframe доступны все функции, необходимые для управления проигрыванием музыки в плейлисте и её прослушивания, но существуют ограничения. Яндекс-музыка и spotify не позволяют слушать песни в виджетах дольше 30 секунд, youtube сжимает качество до 360р.

3. Переименование плейлистов

Для изменения имени плейлиста нажмите кнопку "изменить плейлист" и введите новое название.

4. Удаление

Плейлист можно удалить из списка, нажав на кнопку "удалить плейлист". Перед удалением появится предупреждение.

5. Поиск

Содержимое списков плейлистов начинает фильтроваться сразу после ввода первого символа в поле поиска.

6. Очищение списка плейлистов

С помощью пункта меню "очистить плейлисты" можно очистить все списки плейлистов. Перед очищением появится диалоговое окно.

7.  Сохранение 

Сохранение и загрузка данных производятся автоматически с помощью функций маршализации/демаршализации JAXB и слушателя закрытия окна. 

Технические детали

Основное окно приложения, окна добавления и удаления плейлистов реализрованы с помощью Scene Builder. С помощью файла стилей css  и редактирования свойств css в Scene Builder приложению придан кастомный внешний вид. Scroll pane позволяет умещать любое количество плейлистов в приложении. Приложение работает по схеме Контроллер - Модель - Представление. Кортроллер основного приложения - BakaController, контроллер для окон добавления и удаления плейлистов - AddPlaylistController. Классом модели является Playlist, за представление в основном отвечает BakaPlayerStart, PlaylistList - класс-обёртка для работы JAXB.  Благодаря ObservebleList и Properties внешний вид приложения меняется при действиях пользователя.
